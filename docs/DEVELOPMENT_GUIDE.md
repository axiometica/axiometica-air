## Running Locally

### Quick Start

```bash
# 1. Clone repository
git clone <repo-url>
cd axiometica-air

# 2. Start all services with Docker Compose
docker-compose up -d

# 3. Wait for services to be healthy
docker-compose ps
# Should show all services as "Up"

# 4. Verify API is ready
curl http://localhost:8000/api/health
# Response: {"status": "healthy"}

# 5. Verify database is ready
docker-compose exec postgres psql -U postgres -c "SELECT 1"
# Response: 1

# 6. Access services
# Frontend: http://localhost:3000
# API: http://localhost:8000
# Swagger API docs: http://localhost:8000/docs
# Flower (Celery monitoring): http://localhost:5555
# Neo4j Browser (CMDB): http://localhost:7474
```

### Docker Compose Services

**Services Started:**

```yaml
postgres:            # PostgreSQL 15 (incident/policy/user data)
  - Port: 5432
  - User: postgres / Password: agentic_os / Database: agentic_os

redis:               # Redis 7 (Celery broker + result backend)
  - Port: 6379

neo4j:               # Neo4j 5 Community + APOC (CMDB graph)
  - Port: 7474 (browser UI)
  - Port: 7687 (Bolt)
  - User: neo4j / Password: agentic_os_neo4j

backend:             # FastAPI (2 uvicorn workers)
  - Port: 8000
  - Health check: GET /api/health

celery_worker:       # Celery (2 concurrent workers)
  - Processes incident pipeline tasks asynchronously

flower:              # Celery Flower monitoring UI
  - Port: 5555
  - URL: http://localhost:5555
  - Auth: admin / changeme (configurable via FLOWER_USER / FLOWER_PASSWORD)

frontend:            # React + Vite
  - Port: 3000
  - Hot Module Replacement (HMR) enabled

sentinel_senses:     # eBPF kernel monitor
  - No exposed port (privileged container, bpftrace on host kernel)

watcher_brain:       # Anomaly detection + CMDB discovery
  - No exposed port (communicates via backend API)
```

### Troubleshooting Docker Issues

**Issue: Container won't start**
```bash
# Check logs
docker-compose logs backend

# Common causes:
# 1. Port already in use
netstat -tulpn | grep 8000  # Check port 8000
# Solution: Kill process or change port in docker-compose.yml

# 2. Insufficient disk space
docker system df

# 3. Database not ready
# Solution: Wait for postgres to start, restart other services
docker-compose restart backend
```

**Issue: Database connection refused**
```bash
# Test database connection
docker-compose exec postgres psql -U postgres -c "SELECT 1"

# If fails, check postgres logs
docker-compose logs postgres

# Reinitialize database
docker-compose down -v
docker-compose up -d postgres
docker-compose up -d  # Start other services after postgres ready
```

**Issue: Redis connection error**
```bash
# Test redis connection
docker-compose exec redis redis-cli ping
# Response: PONG

# If fails, check redis logs
docker-compose logs redis
```

## Development Workflow

### Backend Development

**FastAPI + Uvicorn (auto-reload)**

```bash
# Uvicorn watches source files and reloads on changes
docker-compose logs -f backend

# Make changes to backend code
# Uvicorn detects changes and reloads automatically (< 2s)
```

**Adding New API Endpoint:**

1. Create route file in `src/agentic_os/api/routes/`
2. Define Pydantic models for request/response
3. Implement endpoint function
4. Register router in `main.py`

**Example:**
```python
# src/agentic_os/api/routes/custom.py
from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session
from agentic_os.db.database import get_session

router = APIRouter()

class MyResponse(BaseModel):
    message: str
    count: int

@router.get("/my-endpoint", response_model=MyResponse)
async def my_endpoint(db: Session = Depends(get_session)):
    """Get my custom data"""
    count = db.query(WorkflowStateModel).count()
    return MyResponse(message="Success", count=count)
```

**Register in main.py:**
```python
# src/agentic_os/main.py
from agentic_os.api.routes import custom

app.include_router(custom.router, prefix="/api")
```

**Adding New Agent:**

1. Extend `Agent` base class in `agents/`
2. Implement `execute(state: WorkflowState)` method
3. Register in agent registry

**Example:**
```python
# src/agentic_os/agents/custom_agent.py
from agentic_os.agents.base import Agent
from agentic_os.core.context_schema import IncidentWorkflowContext

class MyCustomAgent(Agent):
    name = "MyCustomAgent"

    async def execute(self, context: IncidentWorkflowContext) -> IncidentWorkflowContext:
        """Execute custom logic"""
        result = await self.do_something(context)

        # Add to the reasoning trace
        context.reasoning_trace.append(f"MyCustomAgent: {result}")

        return context

    async def do_something(self, context: IncidentWorkflowContext):
        # Implementation
        return "result"
```

**Register in registry:**
```python
# src/agentic_os/agents/registry.py
def register_all_agents(engine: WorkflowEngine):
    engine.register_agent(MyCustomAgent())
```

**Adding New Component:**

1. Create `.tsx` file in `src/components/`
2. Implement React component (export as default)
3. Import in App.tsx or parent component

**Example:**
```typescript
// src/components/MyComponent.tsx
import React, { useState } from 'react';

interface Props {
  title: string;
  darkMode: boolean;
}

export default function MyComponent({ title, darkMode }: Props) {
  const [count, setCount] = useState(0);
  
  return (
    <div className={darkMode ? 'bg-gray-900 text-white' : 'bg-white'}>
      <h1>{title}</h1>
      <button onClick={() => setCount(count + 1)}>
        Count: {count}
      </button>
    </div>
  );
}
```

**Use in App.tsx:**
```typescript
import MyComponent from './components/MyComponent';

function App() {
  const [darkMode, setDarkMode] = useState(true);
  
  return (
    <>
      <MyComponent title="Test" darkMode={darkMode} />
    </>
  );
}
```

**Adding New API Call:**

```typescript
// src/services/api.ts
export async function getMyData(): Promise<MyResponse> {
  const response = await fetch(`${API_BASE}/my-endpoint`);
  if (!response.ok) throw new Error('Failed to fetch');
  return response.json();
}
```

**Use in Component:**

```typescript
import { getMyData } from '../services/api';

export default function MyComponent() {
  const [data, setData] = useState<MyResponse | null>(null);
  const [loading, setLoading] = useState(false);
  
  useEffect(() => {
    setLoading(true);
    getMyData()
      .then(setData)
      .catch(console.error)
      .finally(() => setLoading(false));
  }, []);
  
  if (loading) return <div>Loading...</div>;
  if (!data) return <div>No data</div>;
  
  return <div>{data.message}</div>;
}
```

## Testing

### Unit Tests

**Location:** `backend/tests/unit/`

```bash
# Run all tests
pytest backend/tests/unit

# Run specific test file
pytest backend/tests/unit/test_repositories.py

# Run with coverage
pytest --cov=agentic_os backend/tests/unit

# Run with verbose output
pytest -v backend/tests/unit
```

**Example Unit Test:**
```python
# backend/tests/unit/test_repositories.py
import pytest
from agentic_os.db.repositories import WorkflowRepository
from agentic_os.core.models import WorkflowState, WorkflowType

def test_save_and_get_workflow(db_session):
    """Test saving and retrieving workflow"""
    repo = WorkflowRepository(db_session)
    
    # Create workflow
    state = WorkflowState(
        workflow_type=WorkflowType.INCIDENT,
        context={"alert_payload": {"type": "high_cpu"}}
    )
    
    # Save
    repo.save(state)
    workflow_id = state.workflow_id
    
    # Retrieve
    retrieved = repo.get(workflow_id)
    
    # Assert
    assert retrieved is not None
    assert retrieved.workflow_id == workflow_id
    assert retrieved.workflow_type == WorkflowType.INCIDENT
```

### Integration Tests

**Location:** `backend/tests/integration/`

```bash
# Run integration tests (needs running database)
```

**Example Integration Test:**
```python
# backend/tests/integration/test_workflow_execution.py
@pytest.mark.asyncio
async def test_incident_workflow_execution(client, db_session):
    """Test full incident workflow"""
    # Submit incident
    response = client.post("/api/workflows/incident", json={
        "severity": "high",
        "type": "high_cpu",
        "resource_name": "api-server",
        "description": "Test incident"
    })
    
    assert response.status_code == 200
    workflow_id = response.json()["workflow_id"]
    
    # Give Celery time to process
    await asyncio.sleep(2)
    
    # Retrieve workflow
    workflow = db_session.query(WorkflowStateModel).filter(
        WorkflowStateModel.workflow_id == workflow_id
    ).first()
    
    # Assert workflow was processed
    assert workflow is not None
    assert workflow.lifecycle_state == "completed"
```

```dockerfile
COPY requirements.txt .

FROM base as final

CMD ["uvicorn", "agentic_os.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**Build:**
